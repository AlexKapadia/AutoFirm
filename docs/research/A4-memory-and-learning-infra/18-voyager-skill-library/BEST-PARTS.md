# BEST-PARTS — Voyager skill library

## ADOPT
- **Verify-before-write for the Experience tier.** Voyager only commits a skill after a
  self-verification step confirms it works. Build implication: AutoFirm's experience/insight tier
  (the ExpeL-style layer, folder 11) writes a distilled insight to durable memory ONLY after a
  SEPARATE judge validates it (mirrors Reflexion's separate-evaluator discipline, folder 10) —
  preventing a bad lesson from poisoning future runs. Ties directly to the WA primitive (folder 12).
- **Procedural memory as retrievable, executable artifacts.** A learned "skill" = a versioned,
  embedding-indexed, reusable unit (for AutoFirm: a parameterized playbook step / code module),
  retrieved by similarity like any other memory. Build implication: AutoFirm's procedural-memory
  store (the CoALA "procedural" axis, folder 02) is concrete — skills are first-class memory records
  with provenance, not prompt boilerplate.
- **Compositionality fights forgetting.** Storing skills as composable code "alleviates catastrophic
  forgetting" — reinforcing the no-fine-tune thesis: durable competence lives in external memory, so
  the hosted model never needs reweighting.

## REJECT / DEFER
- **Reject the embodied auto-curriculum wholesale** — Voyager's exploration curriculum is tuned to an
  open-ended game world; AutoFirm's "curriculum" is the client's real business goals, not
  self-proposed exploration. Adopt the *verified-skill-memory* mechanism; reject the game-specific
  curriculum. (DEFER an internal skill-improvement loop to L2.A4.)

## Build implication (concrete)
Gives L1.A4.3 a third independent primary and supplies the **verify-before-write** rule and the
**procedural-memory-as-executable-skill** design for the L2.A4 experience tier — making "AutoFirm
gets better over time" a memory-write discipline, not a fine-tuning step.
