## Key claims with locators

1. **Reference-model definition (generic + recommendation).** A reference model is a conceptual
   model that is **generic** (intended for reuse across a class of enterprises) and has a
   **recommendation / best-practice character** - it represents how things *should* be done, not
   one firm's as-is. Two defining properties: **universality/generality** (reusable across a domain)
   and **recommendation** (normative best practice). [Survey, definitional section.]

2. **Reference models are reused, not rebuilt.** The economic rationale: a reference model is
   constructed once and reused/adapted by many organizations, lowering modelling cost and raising
   quality - the same "build-once, parameterize-many" logic as C-EPC (source 05) and APQC (04).

3. **Survey scope.** The paper surveys and classifies a substantial set of known business-process
   reference models (the companion Fettke/Loos methodology classifies ~30 reference models),
   applying a multi-dimensional classification framework (criteria include: application domain,
   modelling language used, model size, available evaluations, and documented applications).

4. **Generality rests on a similarity assumption.** Generic reference models assume **similarity
   between enterprises** - either cross-industry or within a sector. The validity of a reference
   model's generality is therefore an empirical claim about that similarity, not an axiom (this is
   exactly why B12 demands the model be PROVEN against a fixed panel, not asserted).
