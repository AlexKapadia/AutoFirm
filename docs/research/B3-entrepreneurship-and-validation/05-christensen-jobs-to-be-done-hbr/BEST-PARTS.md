# BEST-PARTS — Christensen et al., "Jobs to Be Done" (HBR 2016)

## ADOPT
1. **Define the opportunity as a JOB, not a demographic.** AutoFirm's discovery step frames the
   problem hypothesis as a "job to be done" in a circumstance, with explicit **functional / social /
   emotional** dimensions. Build implication: a `JobToBeDone` contract feeding the hypothesis ledger
   — this is *what* to validate; Lean Startup BML (sources 03/04) is *how*.
2. **Hire/fire as a competitive-alternatives check.** Because customers "hire" a solution against
   alternatives (including non-consumption), AutoFirm's validation must enumerate the *current
   solutions the job is hired from today* — this grounds the SAM/SOM (source 10) in real substitution,
   not an abstract category.
3. **Causal job over correlational segmentation** maps directly to CLAUDE SS3.9 (generality, not
   overfitting): design for the causal job, which generalizes across the industry panel, rather than
   fitting a demographic profile that does not.

## REJECT / DEFER
- **Reject as the SOLE JTBD source** — the framework has two competing interpretations (source 09);
  AutoFirm must report both and avoid claiming JTBD is a single settled theory (balance, SS4).
- **Reject over-claiming predictive power** — the milkshake case is illustrative, not validated;
  JTBD informs *hypothesis generation*, and those hypotheses still go through falsifiable testing.
- **Defer the operational job-mapping process** to ODI (source 06), which supplies a measurable
  procedure (desired-outcome statements + opportunity score) the conceptual school lacks.

## Build implication
A `JobToBeDone` typed contract (job statement + circumstance + functional/social/emotional dimensions
+ current alternatives) is the *input* to the validation hypothesis ledger. It makes the venture's
target problem explicit and auditable, and is industry-general by construction.
