# BEST-PARTS — Kotler STP

## ADOPT
- **STP as the mandatory ordered pipeline** for AutoFirm's marketing playbook: `segment -> target -> position` BEFORE any 4P/channel decision. **Build implication:** the L2.B7 playbook is a staged DAG; each stage is gated (you may not enter targeting without a segment set, etc.) — mirrors AutoFirm's own gate-based workflow (CLAUDE.md 4.2).
- **The 5 segmentation-effectiveness criteria (measurable, substantial, accessible, differentiable, actionable)** as a hard validation gate on any segment the engine proposes. **Build implication:** an adversarial test asserts every emitted `Segment` passes all 5 criteria; segments failing (e.g. not measurable on available public data) are rejected fail-closed. This is a tests-with-teeth target (CLAUDE.md 3.6).
- **Targeting evaluation = size/growth x structural attractiveness x fit-to-resources.** **Build implication:** a deterministic scoring function over these axes ranks segments; explainable per CLAUDE.md 3.11 (which axis drove the score).
- **Positioning is relative/perceptual.** **Build implication:** positioning output must reference named competitors and explicit points-of-difference vs. points-of-parity — not an absolute claim. Feeds the brand/messaging stage.

## REJECT
- Reject treating segmentation bases as fixed/universal — *which* base (geo/demo/psycho/behavioral) is industry-dependent and must be chosen from data (generality requirement).
- Reject Kotler as a *sole* source for any load-bearing quantitative claim (textbook tier); corroborate every relied-upon number elsewhere.

## Why
STP gives the playbook its top-level control flow and a testable definition of a "good segment," keeping it general across the industry panel while remaining auditable and explainable.
