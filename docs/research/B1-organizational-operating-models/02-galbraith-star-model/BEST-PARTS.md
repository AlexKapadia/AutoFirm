# BEST-PARTS — Galbraith Star Model

## What AutoFirm ADOPTS

1. **The Star is AutoFirm's org-design contract — for the agent company AND for client companies.**
   Any organization AutoFirm spins up (its own agent org, or a client's company plan) must specify
   all five points, not just structure. A plan that only draws boxes-and-lines is incomplete.
   - **Build implication:** the **L2.ORG** engine and the business-building playbook (L3.BUSINESS)
     emit a 5-field org spec: `{strategy, structure, processes, rewards, people}` per company/unit.

2. **Strategy-first sequencing.** AutoFirm must derive structure FROM the client's strategy/goal,
   never impose a fixed org template. This is the antidote to overfitting (CLAUDE.md §3.9): the org
   shape is a *function* of the declared strategy + industry, not a constant.
   - **Build implication:** function-decomposition (L2.B2) reads the strategy/goal first, then
     selects a structure type (Functional / Geographic / Product / Customer / Front-back) by rule.

3. **Processes = lateral coordination as a first-class artifact.** The "move information up, down,
   across" definition maps to AutoFirm's typed agent message contracts and reconciliation agents —
   the org's *processes* are literally its A2 communication layer.

4. **Rewards/People mapped to agent incentives & capability loading.** For an agent org, "Rewards"
   = the objective/metric each agent optimizes, and "People" = which skill/tool profile is loaded
   into each subagent (progressive disclosure). Misaligned agent objectives = the Star's
   misalignment failure mode (an agent optimizing the wrong metric).
   - **Build implication:** every spawned agent records its objective (reward) and loaded
     skill-profile (people) in the audit trail; an alignment check verifies they match strategy.

## What AutoFirm REJECTS / caution
- **Reject treating the Star as a static one-time design.** Burton & Obel (03) + Donaldson (05)
  show fit decays; AutoFirm must re-run the Star alignment as the goal/industry changes — this is
  the theoretical basis for the recurring **North Star alignment review** (CLAUDE.md §2).
- **Reject importing Galbraith's *specific* structure-type recommendations as universal.** They are
  contingent (see contingency theory, source 05). Use them as a *menu*, selected per industry.

## Quantification for evidence/
- A binary **Star-completeness gate**: 5/5 points specified for every org spec (else FAIL).
- An **alignment score**: count of agent objectives that trace to the declared strategy / total —
  target 100% (mirrors CLAUDE.md "explanations match decisions exactly"). Feeds evidence/ charts.
