# BEST-PARTS — SPIN Selling (Rackham, 1988)

## ADOPT
- **Adopt the SPIN question sequence as the default DISCOVERY script for AutoFirm's complex/B2B sales motion.** It is a directly automatable conversation template (Situation → Problem → Implication → Need-payoff) that an agent can generate per prospect from public firmographic data. *Build implication:* the L2.B8 playbook ships a `discovery_question_generator` keyed by deal size/complexity; for high-value deals it produces S/P/I/N questions, not pitch scripts.
- **Adopt the empirically-grounded rule: in large deals, suppress closing pressure; lead with need-development.** This becomes a guardrail in the autonomous sales agent — for deals above a size/complexity threshold, the agent must not deploy hard-close tactics (which the 35,000-call study shows backfire).
- **Adopt need-state tracking (implied vs. explicit needs)** as a qualification signal feeding the C/A/E "cognition" dimension (source 02) and MEDDIC "Identify Pain" (source 08).

## REJECT
- **Reject SPIN as a UNIVERSAL motion.** Rackham's own scope is the *large* sale; for small/transactional and self-serve (PLG, source 10) sales, the heavy-discovery script is overkill and slows conversion. AutoFirm must SELECT methodology by deal type, not apply SPIN everywhere.
- **Reject the book's prescriptive anecdotes as evidence** — only the behavioural-study findings are citable; the rest is practitioner narrative.

## Why this matters for generality
SPIN parameterizes by deal complexity, not industry: any high-consideration B2B sale (manufacturing capex, fintech enterprise contract, healthcare system procurement — three B12 panel rows) uses the same S/P/I/N skeleton with industry-specific question content. That is generality by parameterization, not overfitting.
